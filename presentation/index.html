"Can't the AI just
figure it out?"

We've heard every objection. Here's why "just use the AI" isn't the answer.

1
"LLMs have massive context windows now"
+
Context windows are RAM, not storage. You can paste 100k tokens today. Tomorrow? Start over. You're paying for the same context every single session. That's not memory that's expensive repetition with a fancy name.
2
"The AI can just read my codebase"
+
Code shows what, not why. Why JWT over sessions? Why that weird edge case handler? Why Prisma over Drizzle? The reasoning isn't in the code. It's in the decisions humans made and those need to be captured somewhere.
3
"We already have Confluence / Notion"
+
Written by humans. Updated by... nobody. Your wiki is 6 months stale. The AI can't tell what's current truth vs outdated artifacts. Structured docs stay fresh because they're tied to actual code changes.
4
"Good code is self-documenting"
+
Clean code tells you what it does. It doesn't tell you about rejected alternatives, architectural tradeoffs, or the bug that led to that workaround. Self-documenting code is a beautiful myth. Context is reality.
5
"Future AI will solve this"
+
Future AI still won't know your specific decisions. It won't know why you chose Redis over Memcached for your use case. Domain-specific context requires domain-specific documentation. That's not changing.

"Why doesn't the webhook retry on 500 errors?"
By design. 500s indicate server issues on your end, so retrying would hammer a broken endpoint. Use 503 for transient errors those retry with exponential backoff. See docs/webhooks.md for retry logic.
